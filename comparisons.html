Reverse Brief — Witnessed

Send message · primary action

CharacteristicCouplingWhat it does
Time Intentional Travel begins at send press. Delivered confirmation arrives after a distinct beat — two separate moments, not one.
Location Intentional Message travels from compose to chat. "Delivered" appears beneath the landed message — not at the top of the screen.
Direction Intentional Upward travel communicates "sent outward." Consistent with incoming messages arriving from outside.
Dynamics Intentional Arc with slight lateral drift. Message settles with overdamped spring — lands, doesn't bounce. Weight communicates consequence.
Modality Intentional UINotificationFeedbackGenerator .success at send — felt in the same register as the physical press.
Expression Default Same animation regardless of message content, length, or emotional register. The system can't read what you sent.

Verdict

Five of six characteristics intentional. The action visibly crosses a boundary — you see it leave, land, deliver. The one default is Expression: the system responds to the action category, not the specific thing you sent.

Reverse Brief — Default

Send message · primary action

CharacteristicCouplingWhat it does
Time Default Message appears at button press. No temporal separation between send and confirmation — one collapsed moment.
Location Violated Toast appears at the top of the screen. The action happened at the bottom. Spatially disconnected — feedback reads as system announcement, not direct response.
Direction Default No directional information. Message appears in list, toast appears elsewhere. No spatial model communicated.
Dynamics Default 80ms fade. Same weight as every other state change. Nothing communicates that this message went anywhere.
Modality Default No haptic or generic light impact — same pattern as tap, scroll, dismiss. No semantic differentiation.
Expression Default No receipt confirmation, no sense of delivery. The action completed locally — it didn't feel like it reached anyone.

Verdict

Location violated; the other five are defaults. "Sent" at the top of the screen produces Heard at best — the system acknowledged the tap. You don't feel the message went anywhere. The action stayed inside the screen.

A confirm button. Decisive fires on press — the haptic and animation begin before your finger lifts. The default fires on release, 80–150ms late. Press and hold on both to feel the gap without haptics.

Reverse Brief — Decisive

Confirm payment · primary action

CharacteristicCouplingWhat it does
Time Intentional Fires at touchDown — animation begins before the finger lifts. The click is the event, not the confirmation of it.
Dynamics Intentional Compress (0.94) → overshoot (1.06) → settle. Heavier spring than navigation or dismiss — differentiated by action weight.
Modality Intentional UIImpactFeedbackGenerator (intensity 0.8) at touchDown, not touchUpInside. 80–150ms earlier = "this clicked" vs. "this confirmed."
Direction Intentional Compresses in, overshoots out — the direction reversal communicates "it took your input and pushed back."
Location Default Button clicks in place — appropriate for this pattern.
Expression Default Same snap regardless of how decisively or hesitantly the button was pressed.

Verdict

Time and Modality are load-bearing. The animation shape is close to the default — the entire felt difference is when it fires. TouchDown vs. touchUpInside is the whole thing.

Reverse Brief — Default

Confirm payment · primary action

CharacteristicCouplingWhat it does
Time Violated Fires on mouseup / touchUpInside — the response is 80–150ms after the physical action. The click already happened; the button is catching up.
Dynamics Default Generic spring — same weight as navigation, dismiss, toggle. No differentiation by action consequence.
Modality Default Haptic at animation end (or none) — fires after the felt click moment, not with it.
Direction Default Compresses and returns — no overshoot. Soft deceleration to rest.
Location Default Same — button stays in place.
Expression Default Same for all presses.

Verdict

Time is violated. The button animates after you've already committed — you watch the system catch up to a decision you already made. The animation follows the action instead of being the action.

Grounded vs. Default

Deleting a document. Grounded makes deletion feel consequential — the item compresses at commit before it leaves, then exits with weight. The default uses the same animation as closing a tab.

Reverse Brief — Grounded

Destructive action · delete document

CharacteristicCouplingWhat it does
Time Intentional Two distinct moments: compress at commit (something real is happening), then exit. The pause before it leaves is the acknowledgment.
Dynamics Intentional Compress (0.97) at commit. Weighted sink exit — item falls slightly and fades, not slides. Horizontal exit reads as archive; downward reads as gone.
Modality Intentional Heavy impact haptic at commit — same weight register as the consequence of permanent deletion.
Location Intentional Item exits spatially — the space it occupied collapses after, not simultaneously. The absence is felt.
Direction Default Exits right — clear, but not semantically loaded by direction.
Expression Default Same weight for all items — a frequently-used document and a stale draft get identical animations.

Verdict

The compress at commit is load-bearing. It's the moment that communicates "something real happened before this left." Without it, the deletion is just a dismiss — Featherweight applied to a Grounded action.

Reverse Brief — Default

Destructive action · delete document

CharacteristicCouplingWhat it does
Time Default Single moment — item disappears. No commit beat, no pause before exit. Action and consequence collapse into one.
Dynamics Default Same fade/scale as archiving or tab-closing. Deletion has no more felt weight than dismissing a notification.
Modality Default Light or no haptic — same as selection. Nothing communicates permanence or consequence.
Location Default Item fades in place — no exit path, no directional consequence, space collapses immediately.
Direction Default No direction — opacity to zero. The item just stops existing.
Expression Default Same for all. Consequence is invisible.

Verdict

Default delete is Featherweight at best — it feels reversible even when it isn't. Five of six characteristics at default. The user doesn't feel that something real happened; they feel like they dismissed something.

Punctuated vs. Default

A task completion notification. Punctuated has a felt beat — fast attack, slow decay. The default slides in as one motion with no peak, no boundary between before and after.

Reverse Brief — Punctuated

Task completion · success notification

CharacteristicCouplingWhat it does
Time Intentional Two distinct phases: attack (140ms), decay (380ms). Haptic fires at peak — after the attack, before the decay. A felt boundary.
Dynamics Intentional Asymmetric: fast overshoot attack (ease-out-back), slow settling decay (ease-out). The downbeat shape. You feel before and after.
Modality Intentional Medium impact haptic at the peak — marks the boundary moment, not the appearance.
Location Intentional Appears proximate to the action — below the completed task, not system-level top-of-screen.
Direction Default Resolves in place — no directional travel. Could carry spatial meaning.
Expression Default Same animation for all completions — no differentiation for overdue tasks or streaks.

Verdict

Two animations, not one. A single curve can't produce a felt peak — there's no definable maximum in a continuous ease. Attack and decay must be separate, with the haptic fired between them. That gap is the downbeat.

Reverse Brief — Default

Task completion · success notification

CharacteristicCouplingWhat it does
Time Default Single animation — notification appears and settles. No temporal separation, no felt peak. One moment, not two.
Dynamics Default Constant ease-out — decay only, no attack. The notification arrives; it doesn't land. No downbeat.
Modality Default Haptic at appearance — before any visual peak, so disconnected from what it's meant to mark.
Location Default Generic entry — slides from below regardless of where the action was.
Direction Default Same entry direction for all notifications.
Expression Default Same for all completions. Same slide for a routine task and a long-overdue one.

Verdict

Default notification has decay but no attack — it slides in without a boundary. You see it appear. You don't feel it land. The fast overshoot before the slow settle is what creates the felt moment; a single ease produces neither.
